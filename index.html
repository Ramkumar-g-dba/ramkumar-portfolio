Project 03 · Apr 2026 – Present
Snowflake Cost Optimization
Query Analysis → SQL Optimization → Warehouse Tuning → Resource Monitor → Cost Dashboard
🎯 Problem Statement
Identify and reduce unnecessary Snowflake credit usage through query optimization, warehouse right-sizing, partition pruning, and automated cost monitoring using ACCOUNT_USAGE views.
  • Identified inefficient queries causing full table scans (SELECT *, no WHERE filters)
  • Optimized SQL with filters enabling partition pruning — reduced data scanned by ~85%
  • Right-sized warehouse (SMALL → X-SMALL) — 50% credit reduction per query
  • Reduced auto-suspend from 5 min → 60 sec — eliminated idle compute cost
  • Implemented Resource Monitor with credit limits and automated alerts at 75% & 90%
  • Built cost monitoring dashboard using SNOWFLAKE.ACCOUNT_USAGE views
✅ Result

~30% overall credit reduction · 85% less data scanned · Automated cost alerts · Real-time usage dashboard
